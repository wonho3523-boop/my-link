# 마이링크 (MyLink) - 화면 와이어프레임 (Wireframe)

서비스의 핵심이 되는 두 가지 화면(퍼블릭 페이지, 관리자 대시보드)에 대한 화면 구성을 설계합니다.

---

## ⚙️ 공통 UI/UX 정책 (합의 완료)
- **PC 레이아웃 (반응형):** 넓은 모니터 환경에서도 퍼블릭 페이지는 모바일 화면 비율의 **가운데 고정된 컨테이너 영역** 안에 콘텐츠를 표시합니다.
- **공유 피드백 (UX):** 링크 복사 등 사용자 액션 성공 시, 딱딱한 기본 알람(alert) 대신 화면에 부드럽게 나타나는 **토스트 메시지(Toast Message: "클립보드에 복사되었습니다")**를 사용합니다.

---

## 1. 퍼블릭 페이지 (방문자 뷰)

방문자가 접속했을 때 보게 되는 모바일 환경 기준의 화면 레이아웃입니다. 직관적이고 깔끔한 UI를 지향합니다.

### 📊 Mermaid UI (구조 다이어그램)
```mermaid
flowchart TD
    subgraph 퍼블릭_화면 [Public Page Component Layout]
        direction TB
        Header([우측 상단: 공유 버튼])
        Profile((프로필 이미지))
        UserInfo[아이디 @username\n소개글 바이오]
        
        subgraph 링크_목록 [동적으로 렌더링되는 링크 리스트]
            direction TB
            Link1[📺 유튜브 채널]
            Link2[💼 포트폴리오 웹사이트]
            Link3[✉️ 비즈니스 문의 이메일]
        end
        
        Footer>하단 로고: Powered by MyLink 진입점]
        
        Header --> Profile --> UserInfo --> 링크_목록 --> Footer
    end
```

## 2. 관리자 대시보드 (소유자 뷰)

링크 소유자가 로그인 후 진입하는 설정 페이지입니다. 링크 관리 및 수정 기능이 중심이 됩니다.

### 📊 Mermaid UI (구조 다이어그램)
```mermaid
flowchart TD
    subgraph 관리자_대시보드 [Admin Dashboard Layout]
        direction TB
        GNB[Global Navigation Bar: 로고 / 내 프로필]
        
        subgraph 메인_컨텐츠 [2-Column Layout: 에디터 및 실시간 미리보기]
            direction LR
            
            subgraph 좌측_에디터 [Left: 제어 패널]
                direction TB
                LNB[메뉴 탭: Links / Appearance / Analytics / Settings]
                AddBtn{{+ 새로운 링크 추가 버튼}}
                subgraph 링크_리스트 [개별 링크 컨트롤러 리스트]
                    LinkCards[제목, URL 수정 / 토글 / 드래그 핸들]
                end
                LNB --> AddBtn --> 링크_리스트
            end
            
            subgraph 우측_미리보기 [Right: Live Preview 패널]
                Mockup[스마트폰 형태의 가상 렌더링 화면\n퍼블릭 페이지 뷰 실시간 반영]
            end
            
            좌측_에디터 --- 우측_미리보기
        end
        
        GNB --> 메인_컨텐츠
    end
```
